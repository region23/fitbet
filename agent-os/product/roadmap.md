# Product Roadmap

## Implementation Status

**Phase 1 MVP:** ✅ Completed (~95%)
- Core challenge flow fully implemented
- LLM integration with multimodal vision analysis
- Automated check-in windows and reminders
- Democratic Bank Holder voting system
- Winner calculation and final results

**Outstanding:** Minor UX improvements and testing

---

## Phase 1: MVP

Минимальный функционал для запуска первого сезона:

### Создание и управление челленджем
- [x] Создание челленджа в групповом чате (`/create`)
- [x] Настройка параметров: длительность (6/12 мес), ставка, порог дисциплины, политика пропусков
- [x] Назначение Bank Holder после завершения онбординга участников через демократическое голосование

### Онбординг участников
- [x] Вступление через кнопку "Join" в групповом чате
- [x] Приватный онбординг в личке бота:
  - [x] Выбор трека (Cut/Bulk)
  - [x] Ввод стартовых данных (вес, талия, рост)
  - [x] Загрузка стартовых фото (4 ракурса: анфас, профиль слева, профиль справа, со спины)
  - [x] Постановка цели с рекомендациями по формулам (BMI, WHtR) + LLM-валидация
  - [x] Выбор 2-3 process commitments

### Финансы
- [x] Отметка "Я оплатил" от участника
- [x] Подтверждение оплаты Bank Holder'ом
- [x] Статусы участника: `joined_pending_payment` → `active`

### Чек-ины
- [x] Автоматическое расписание окон чек-ина (каждые 14 дней, окно 48 часов)
- [x] Сбор данных: вес, талия, 4 фото (анфас, профиль слева, профиль справа, со спины)
- [x] Напоминания за 12 часов до закрытия окна
- [x] Публикация списка не сдавших в групповой чат
- [x] Учёт пропусков и дисциплины
- [x] **БОНУС:** LLM-анализ прогресса с мультимодальным vision-анализом фото (рекомендации по питанию, тренировкам, мотивация)

### Финал
- [x] Расчёт победителей по формуле (70% достижение цели + 30% дисциплина)
- [x] Публикация результатов в чат
- [x] Инструкции выплат для Bank Holder
