# Product Roadmap

## Phase 1: MVP

Минимальный функционал для запуска первого сезона:

### Создание и управление челленджем
- [ ] Создание челленджа в групповом чате (`/create`)
- [ ] Настройка параметров: длительность (6/12 мес), ставка, порог дисциплины, политика пропусков
- [ ] Назначение Bank Holder после завершения онбординга участников. Выбирается среди участников челенджа.

### Онбординг участников
- [ ] Вступление через кнопку "Join" в групповом чате
- [ ] Приватный онбординг в личке бота:
  - Выбор трека (Cut/Bulk)
  - Ввод стартовых данных (вес, талия, рост)
  - [x] Загрузка стартовых фото (4 ракурса: анфас, профиль слева, профиль справа, со спины)
  - [x] Постановка цели с рекомендациями по формулам (BMI, WHtR) + LLM-валидация
  - Выбор 2-3 process commitments

### Финансы
- [ ] Отметка "Я оплатил" от участника
- [ ] Подтверждение оплаты Bank Holder'ом
- [ ] Статусы участника: `joined_pending_payment` → `active`

### Чек-ины
- [ ] Автоматическое расписание окон чек-ина (каждые 14 дней, окно 48 часов)
- [x] Сбор данных: вес, талия, 4 фото (анфас, профиль слева, профиль справа, со спины)
- [ ] Напоминания за 12 часов до закрытия окна
- [ ] Публикация списка не сдавших в групповой чат
- [ ] Учёт пропусков и дисциплины

### Финал
- [ ] Расчёт победителей по формуле (цель + дисциплина)
- [ ] Публикация результатов в чат
- [ ] Инструкции выплат для Bank Holder

## Phase 2: v1 (Post-Launch)

Улучшения UX и "вау-контент":

### Визуализации
- [ ] Индивидуальный дашборд в виде сообщения с кнопками
- [ ] График веса (тренд за весь период)
- [ ] График талии
- [ ] Фото-лента всех чек-инов
- [ ] Таймлапс/анимашка из фото (слайд-шоу/GIF)

### Карточки
- [ ] Финальная карточка "start vs end" с метриками
- [ ] Карточка чек-ина (вес, талия, номер чек-ина, статус)

### Дополнительно
- [ ] Опциональный %жира по BIA-весам (как тренд)
- [ ] Режимы приватности фото (скрыто в группе / превью / спойлер)
- [ ] Возможность замены чек-ина в течение окна (1 раз)
